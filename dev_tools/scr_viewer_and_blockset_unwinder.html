    Drop an .SCR file here.

    This takes the raw data of a Super Hi Res .SCR file and paints it onto a canvas at 2x scale. It handles palettes and SCBs fine with two exceptions: It doesn't handle "fill mode" and it doesn't handle the 640-pixel flag.

  2. Drop a processed blockset here file here to unwind it.

    This takes a blockset that's in the Super Merryo Trolls single-stripe format (essentially a 20 x (16*(16*12)) pixel column of blocks, at two pixels per byte), cuts the stripe up into blocks, and copies the blocks into an .SCR file, making a 320x192 pixel template (16x12 blocks at 20x16 pixels each) suitable for editing in a paint program on the Apple IIgs. It basically un-pre-processes the block data.

    It then renders the resulting .SCR file. Note that the palette and SCB data in this .SCR file is inherited from the "original SCR file" above. (A pre-processed block stripe has no palette data of its own.)
